## 📌 目標: 在 WSL 中透過命令列操作 Quartus 工具

由於個人不喜歡 Quartus 的 UI 介面，且 Quartus 作為集合各種功能的 IDE，太過複雜笨重。
因此透過在 WSL 原有自行搭建的開源 Verilog 編譯模擬環境下，加入透過命令列操作 Quartus 的工具，以實現編譯與燒錄進友晶 (Terasic) 的 DE0-Nano FPGA 開發版。


## 🛠️ Quartus 核心命令列工具 (CLI)

```bash
export PATH=$PATH:/mnt/c/intelFPGA_lite/21.1/quartus/bin64/
```

在 WSL 中操作 Quartus，主要是透過呼叫 Windows 端的執行檔（副檔名為 `.exe`）。如果你的系統路徑已設定好，常見的工具如下：

| 工具名稱 | 功能說明 | 常用指令範例 |
| --- | --- | --- |
| **quartus_map** | 分析與分析 (Synthesis) | `quartus_map <project_name>` |
| **quartus_fit** | 佈局與佈線 (Place & Route) | `quartus_fit <project_name>` |
| **quartus_asm** | 產生燒錄檔 (.sof / .pof) | `quartus_asm <project_name>` |
| **quartus_tan** | 時序分析 (Timing Analysis) | `quartus_tan <project_name>` |
| **quartus_pgm** | 裝置燒錄 (Programming) | `quartus_pgm -m jtag -o "p;project.sof"` |


build.sh
```bash
#!/bin/bash

PROJECT_NAME="fpga_project"

echo "Starting Compilation..."

quartus_sh.exe --flow compile $PROJECT_NAME
```

program.sh
```bash
#!/bin/bash

SOF_FILE="output_files/fpga_project.sof"

echo "Checking for USB-Blaster..."
quartus_pgm.exe -l

echo "Programming DE0-Nano..."
# -m jtag: 使用 JTAG 模式
# -c: 指定硬體名稱 (通常是 USB-Blaster)
# -o: 指令 "p" 代表 program (燒錄)
quartus_pgm.exe -m jtag -c "USB-Blaster" -o "p;$SOF_FILE"
```

## 🔌 在 WSL 中操作 DE0-Nano (USB-Blaster)

這是最棘手的部分，因為 WSL 預設無法直接存取 Windows 的 USB 裝置。你需要在 Windows 安裝 **usbipd-win**，並在 WSL 中掛載 USB-Blaster：

1. **Windows 端執行**: `usbipd bind --hardware-id <VID:PID>`
2. **Windows 端執行**: `usbipd attach --wsl --hardware-id <VID:PID>`
3. **WSL 端檢查**: 使用 `lsusb` 確認是否看到 Altera USB-Blaster。
